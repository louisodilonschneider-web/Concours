# Bibliothèque EPS

Bibliothèque EPS est une application web HTML autonome dédiée à la consultation de citations et de références utiles aux épreuves de CAPEPS et d’Agrégation en EPS [file:1]. Le projet est conçu pour fonctionner directement depuis un fichier `index.html`, afin d’être compatible avec GitHub Pages [file:1].

## Présentation

L’interface propose deux vues principales : **Citations** et **Bibliographie** [file:1].  
Elle permet de rechercher, filtrer et organiser des contenus autour de l’enseignement de l’EPS, des enjeux de formation, du corps, des paradigmes scientifiques et d’autres notions liées au champ EPS [file:1].

## Fonctionnalités

- Recherche par citation, auteur, thème ou référence [file:1].
- Filtres par épreuve, acteurs, thèmes, bornes et précisions [file:1].
- Affichage des citations sous forme de cartes [file:1].
- Consultation de la bibliographie par auteur [file:1].
- Copier une citation rapidement [file:1].
- Mode édition protégé par code PIN [file:1].

## Organisation du projet

Le projet repose sur une page unique contenant :
- un en-tête avec navigation et recherche [file:1],
- une section dédiée aux citations avec filtres latéraux [file:1],
- une section bibliographique avec recherche et tri alphabétique [file:1].

Le fichier principal du projet doit s’appeler **`index.html`** pour fonctionner correctement sur GitHub Pages [file:1].  
Si tu as renommé le dossier ou la page en `index.html`, ce nom doit être conservé tel quel dans l’arborescence du dépôt [file:1].

## Utilisation

1. Ouvrir le fichier `index.html` dans un navigateur [file:1].
2. Utiliser la barre de recherche pour trouver une citation ou un auteur [file:1].
3. Appliquer les filtres pour affiner les résultats [file:1].
4. Aller dans l’onglet **Bibliographie** pour parcourir les auteurs [file:1].
5. Activer le mode édition si besoin de modifier une entrée [file:1].

## Mise en ligne sur GitHub Pages

Pour que le site fonctionne sur GitHub Pages, le fichier d’entrée doit être nommé **`index.html`** [file:1].  
Il suffit ensuite de déposer le contenu du projet dans le dépôt GitHub puis d’activer GitHub Pages sur la branche contenant ce fichier [file:1].  
Aucune configuration supplémentaire n’est visible dans le fichier analysé, donc la structure actuelle semble adaptée à un déploiement simple [file:1].

## Personnalisation

Le fichier contient le HTML, le CSS et le JavaScript dans un seul document [file:1].  
Les styles sont définis directement dans la page via des variables CSS, ce qui facilite les ajustements visuels [file:1].  
Les données des citations et références sont intégrées dans un script interne, donc les ajouts ou corrections se font directement dans le fichier source [file:1].
